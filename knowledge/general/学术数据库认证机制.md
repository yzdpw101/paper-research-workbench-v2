# 学术数据库认证机制

## IEEE Xplore

### 认证方式

| 方式 | 机制 | Cookie 持久性 |
|------|------|:---:|
| **机构 IP 认证** | 基于 IP 地址识别机构 | 无持久 cookie |
| **机构登录 (SAML/CARSI)** | 重定向到机构 IdP | Session cookie |
| **个人账户** | IEEE Account (email + password) | 持久 cookie |
| **VPN/代理** | 通过机构网络 | 取决于 VPN |

### 关键发现

> **机构网络环境：** 打开 IEEE/万方自动完成机构登录（IP 识别），无需任何操作。
> 
> **非机构网络环境：** 登录后产生 Cookie，但隔一段时间会断登录态。重新登录时**不需要重新填写账号密码**（凭据可自动填充），但**仍然需要手动点击**完成机构登录流程（如 CARSI 重定向、SAML 跳转、点击 "Continue" 等交互步骤）。
> 
> 这意味着：自动填充凭据能省掉输入步骤，但不能实现"全自动无人值守登录"。

### 下载机制

IEEE PDF 下载 URL 格式：
```
https://ieeexplore.ieee.org/stampPDF/getPDF.jsp?tp=&arnumber=<n>
```

这个 URL 可通过 `context.request.fetch()` 直接获取 PDF 字节流。

## 万方数据

### 认证方式

| 方式 | 机制 | Cookie 持久性 |
|------|------|:---:|
| **机构登录** | CARSI 或机构代理 | Session cookie |
| **个人账户** | 万方账号 | 持久 cookie |
| **机构 IP** | IP 识别 | 无持久 cookie |

### 下载机制

- **硕博论文：** 新标签页打开 → 倒计时 → 点击"点击此处" → 触发下载
- **期刊论文：** 直接下载按钮 → `page.on('download')` → 保存

## 对我们项目的影响

### 机构网络用户（最常见）
- storageState 作用有限（IP 认证无 cookie）
- 但 persistent context 的 localStorage/IndexedDB 可能缓存一些状态
- **不需要额外登录处理**

### 非机构网络用户
- **方案 A：加密凭据 + 自动登录**
  - 用户首次手动登录
  - credential-vault 存储凭据
  - 后续自动填充登录表单
  - 登录后保存 storageState（session 有效期内复用）
- **方案 B：storageState 持久化**
  - 登录一次 → 保存 storageState
  - 后续复用（直到 cookie 过期）
  - 过期后提示重新登录

### 推荐混合策略（修正版）

```
1. 加载 storageState
2. 检测登录状态（page evaluate）
3. 如果是机构网络 → 已自动登录（IP 识别），无需任何操作 ✅
4. 如果是非机构网络：
   a. 如果 storageState 有效（Cookie 未过期）→ 直接操作 ✅
   b. 如果登录态过期：
      - 从 credential-vault 读取凭据
      - 自动填充账号密码（省去输入）
      - 用户手动点击完成登录流程（CARSI/SAML 重定向等）
      - 登录成功后保存 storageState
   c. 如果凭据不存在 → 完全手动登录
```

> ⚠️ **重要修正：** 非机构网络 CARSI 登录（CARSI/SAML）无法完全自动化——重定向、IdP 选择、可能的验证码等步骤需人工完成。auto-login 模块的目标是"自动填充凭据 + 辅助完成流程"，而非"全自动无人值守登录"。
