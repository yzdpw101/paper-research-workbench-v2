# 优化计划

## 设计原则

- 搜索永远 launch（headless），不依赖 config 的 mode 设置
- 下载走 launch 或 CDP（由网络环境决定）
- 登录与下载分离：`wf-carsi-login.js` 管登录，`wf-download.js` 管下载
- 凭据录入有两个入口：初始化网页 + 下载前补救

---

## 1. 初始化网页（`credential-page.js` 改造）

**现状**：`credential-page.js` 是浏览器内 HTML 表单，功能单一。

**改为**：综合初始化页面，包含三块：

|  模块 | 内容  |
| ---|--- |
|  环境检测 | Node.js 版本、Playwright 安装状态、可用浏览器列表。缺什么给出安装指令  |
|  凭据录入 | IEEE / 万方账号密码 + 机构名。非必须，跳过则只能走机构网络  |
|  浏览器选择 | Chrome / Edge / Firefox。选 Firefox 提示"仅支持机构网络"  |

**产出**：`.state/config.json` + `.state/credentials.json.enc` + `.state/.setup-done`

---

## 2. 网络确认页面

每次加载 skill 后，如果任务包含"下载"，弹出选择：

```
你当前的网络环境？
○ 机构网络（校园网/VPN） → 下载走 launch 模式
○ 非机构网络 → 检查凭据 → 没有则提示录入 → 下载走 CDP 模式（Chrome/Edge）
```

选择结果写入 `.state/config.json` 的 `networkMode` 字段，后续脚本读取。

---

## 3. 搜索脚本（`wf-search.js` / `ieee-search.js` / `ieee-detail.js` / `ieee-figures.js`）

**改动**：`launch({ headless: true })` 改为强制 launch 模式，忽略 config 的 cdp。

```js
// 搜索永远 launch，不走 CDP
const { browser, page } = await launch({ headless: true, mode: 'launch' });
```

无登录逻辑（已实现）。

---

## 4. 下载脚本（`wf-download.js`）

新增参数：

|  参数 | 说明  |
| ---|--- |
|  `--mode <launch\|cdp>` | 覆盖 config 的 mode  |
|  `--cdp-port <n>` | CDP 端口（默认 9222）  |

流程：

```
1. 读取 --mode 或 config → 决定 launch 还是 CDP
2. launch 模式 → Playwright 直接控制下载目录 → 完毕
3. CDP 模式 → 连接 Chrome → 下载到 Chrome 默认目录 → 轮询 → 复制到 --save-as
```

**不含登录逻辑**。登录由 `wf-carsi-login.js` 单独处理。

---

## 5. 登录脚本（`wf-carsi-login.js`）

保持现有导出：`checkStatus(page)` / `login(page, creds, opts)`。

CLI 用法不变：
```
PAPER_MASTER_KEY=<key> node scripts/wf-carsi-login.js --port=9222
```

---

## 6. 完整下载流程（agent 视角）

### 机构网络
```
node scripts/wf-download.js --q "..." --type thesis --idx 0 --save-as "..." --mode launch
```

### 非机构网络
```
# Step 1: 启动 CDP Chrome
scripts/open-chrome-cdp.bat

# Step 2: 登录（首次或 Cookie 过期时）
PAPER_MASTER_KEY=<key> node scripts/wf-carsi-login.js --port=9222

# Step 3: 下载
PAPER_MASTER_KEY=<key> node scripts/wf-download.js --q "..." --type thesis --idx 0 --save-as "..." --mode cdp
```

> Cookie 有效时 Step 2 可跳过，`wf-carsi-login.js` 会检测到已登录并直接退出。
